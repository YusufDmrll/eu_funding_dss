# Gillis Feedback Questions

Use these questions after testing several realistic project descriptions. Short notes and concrete examples are enough.

## Match quality

1. Were the top funding calls relevant to the project idea?
2. Which result was most useful, and why?
3. Which result felt irrelevant or misleading?
4. Were any important funding areas or calls missing?
5. Did the shortlist contain the right number of results, or should it be more selective?

## Interpretation

6. Did **Strong match**, **Worth reviewing**, and **Needs more detail** feel clear and credible?
7. Did any review status seem too confident or too cautious?
8. Was the distinction between relevance and overall fit understandable?

## Explanations and actions

9. Did **Why it matters** explain the real connection between the project and the call?
10. Were any explanations generic, repetitive, or based on weak wording?
11. Was the recommended next action practical for deciding whether to investigate the call?
12. Did the main caution identify the most important limitation?
13. Did **What supports the fit**, **What to clarify**, and **Next steps for this call** help you decide what to do next?

## PDF and workflow

14. Was the PDF useful as a screening summary to save or share?
15. Was anything important missing from the PDF?
16. Was any information unnecessary or too detailed?
17. Could you move from project description to shortlist without needing technical guidance?

## Product direction

18. Which themes should be expanded first?
19. Should the tool feel more consultant-facing, with deeper screening detail, or more client-facing, with a simpler decision summary?
20. Which part would save the most time in repeated company use?
21. What would prevent you from trusting or regularly using the current version?

## Useful feedback format

For each tested project, record:

- Project description used
- Most relevant result
- Irrelevant or questionable result
- Missing funding area
- Status or explanation concern
- PDF feedback
- Suggested improvement
